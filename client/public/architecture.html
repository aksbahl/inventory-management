Data Flow — Filter Selection to Rendered UI
1
User selects a filter in the sidebar
FilterBar dropdowns are bound via v-model to reactive refs inside the useFilters() singleton composable. All 7 views share the exact same state object — no prop drilling.
2
watch() triggers a data fetch
Each view watches the shared filter refs. On change, it calls getCurrentFilters() which translates UI-friendly names to API query param names, then calls the relevant api.js function.
getCurrentFilters() → { warehouse: 'Tokyo', category: 'sensors', month: '2025-03' }
3
Axios sends the HTTP request
api.js builds a query string from the filter object and fires the request. Parameters with value 'all' are omitted, meaning no filter is applied for that dimension.
GET http://localhost:8001/api/orders?warehouse=Tokyo&category=sensors&month=2025-03
4
Backend filters in-memory data and validates
apply_filters() chains warehouse → category → status predicates. filter_by_month() handles YYYY-MM or Q#-YYYY quarter formats. Results are validated by Pydantic before serialization.
apply_filters(orders, warehouse='Tokyo', category='sensors') filter_by_month(items, month='2025-03')
5
Vue reactive update re-renders the UI
Response is assigned to a raw ref (e.g., allOrders.value). Computed properties for filtered lists, KPI sums, and chart data recalculate automatically. Vue diffs and patches only the affected DOM nodes — no manual updates needed.
Filter System
Time Period
?month=
YYYY-MM (2025-01 through 2025-12) or quarter Q1-2025 through Q4-2025. Omit for all dates. Not supported by inventory endpoints.
Warehouse / Location
?warehouse=
San Francisco, London, Tokyo. Case-sensitive exact match against the warehouse field on orders and inventory items.
Category
?category=
Circuit Boards, Sensors, Actuators, Controllers, Power Supplies. Case-insensitive match applied to both orders and inventory.
Order Status
?status=
Delivered, Shipped, Processing, Backordered. Only applies to /api/orders and /api/dashboard/summary. Case-insensitive.
API Endpoints
GET /api/inventory
Stock levels, quantities, reorder points for all SKUs
?warehouse   ?category
GET /api/inventory/{id}
Single inventory item detail
GET /api/orders
Order list with nested line items, status, and delivery dates
?warehouse   ?category   ?status   ?month
POST /api/orders
Create a new order record
GET /api/dashboard/summary
KPI aggregates — fulfilled count, fill rate, revenue, avg processing time
?warehouse   ?category   ?status   ?month
GET /api/demand
Demand forecasts with trend direction per SKU (no filters)
GET /api/backlog
Unfulfilled orders with shortage quantities and purchase order linkage flags
GET /api/restocking/recommendations
Items with increasing demand trend, recommended restock quantities and costs
GET /api/spending/summary
Total procurement, operational, labor, and overhead costs with YoY change %
GET /api/spending/monthly
Month-by-month spending breakdown by cost type
GET /api/spending/categories
Spending totals aggregated per product category
GET /api/spending/transactions
Recent transaction ledger entries
GET /api/reports/quarterly
Quarterly aggregated performance statistics
GET /api/reports/monthly-trends
Month-over-month trend data for the Reports view
POST /api/purchase-orders
Submit a supplier purchase order linked to a backlog item
